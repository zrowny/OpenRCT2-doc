root allOf item 5 then properties buildMenuPriority
Type: integer

For rides that do no list their subtypes separately, this number describes the priority order for which subtype should show for the generic ride type in the build menu. Of all the subtypes that are available and researched, whichever has the highest buildMenuPriority will show as representative of the generic ride type.

Value must be greater or equal to 0 and lesser or equal to 255

Type: boolean Default: false

root allOf item 5 then properties playDepartSound
Type: boolean Default: false

Flagged if the ride plays a departure sound when departing the station. depending on sound_range setting, plays Tram or Train departing sound.

root allOf item 5 then properties playSplashSound
Type: boolean Default: false

Flagged if the ride should play a splashing sound on down to flat elements

root allOf item 5 then properties playSplashSoundSlide
Type: boolean Default: false

Flagged if the ride should play a splashing sound when entering a water channel, for water coasters. Has no effect if playSplashSound is enabled.
Note: Internally, water channel track is coded as "covered" track, so if this flag is set for a ride running on a track that supports covered pieces, it will play a splash sound when entering a covered section of track.

root allOf item 5 then properties hasShelter
Type: boolean Default: false

Flagged if the ride is covered (for example, monorail cars are covered)
Note that there are some ride types in vanilla RCT2 that seem to have this bit set illogically. Pickup-trucks did not have this set, and the uncovered ski lift cars did have these set. These have been changed in OpenRCT2 to make more sense.

root allOf item 5 then properties limitAirTimeBonus
Type: boolean Default: false

Flagged if the ride should have a hard cap on how much bonus it gets from airtime. This is only set for heartline-twister coasters, and makes it so that a max of ~2 seconds of airtime can give an excitement bonus.
